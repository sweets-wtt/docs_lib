# docs_lib

> 通用型项目文档与技选模板库，为 AI Agent 搭建项目框架提供完整的参考依据。

## 这是什么

本仓库包含两份核心文档，分别解决项目初始化阶段的两个关键问题：

- **`project_documentation_structuring.md`**：项目文档结构标准。定义根目录四份核心文档（`README.md`、`AGENTS.md`、`SPEC.md`、`PROMPT.md`）与三份配置文件（`.gitattributes`、`.gitignore`、`.editorconfig`）的职责、模板与交叉引用规则；包含四文档职责边界、Agent 文档生成工作流与质量检查清单。

- **`project_technology_selection.md`**：项目技术选型定义。覆盖运行环境、Monorepo 工具链、共享包与 API 契约、质量与安全门禁、本地基础设施、前端技术栈（双端通用 + Web SPA + 微信小程序）、后端技术栈；每项技术包含类别、职责、用途与红线约束。

## 为什么需要它

新启动项目时，团队常常面临这些问题：

- "技术选型该怎么定？哪些工具该用、哪些不该用？"
- "项目文档该怎么组织？各文档写什么、不写什么？"
- "各文档之间怎么避免重复和冲突？"
- "Agent 参与项目时该遵循什么规则？"

这两份文档直接给出答案。它们不是抽象的占位符集合，而是基于行业最佳实践的具体定义——每个文件条目包含职责说明、标准模板结构与参考标准链接；每项技术包含类别、职责、用途与红线约束。

## 快速开始

### 方式一：Agent 自动搭建

1. 检索 `project_technology_selection.md` 确定项目所需的技术栈范围
2. 检索 `project_documentation_structuring.md`，按「Agent 文档生成工作流」四阶段（准备 → 分析与对照 → 生成 → 质量验证）生成根目录文档与配置文件

### 方式二：手动参考

1. 打开 `project_technology_selection.md`，查阅项目所需的技术章节
2. 打开 `project_documentation_structuring.md`，按各文件的「标准模板结构」创建文档
3. 确保各文档之间的交叉引用链接指向正确（参见「四文档职责边界」与「交叉引用规则」）

## 设计原则

- **具体而非抽象**：模板中使用真实的文件名、命令示例、章节标题，而非空泛的占位符
- **职责边界清晰**：每个文件和目录有明确的职责范围，避免内容重复（参见「四文档职责边界」）
- **交叉引用完整**：文档之间通过链接相互指引，形成知识网络而非孤立岛屿（参见「交叉引用规则」）
- **参考标准权威**：各文件条目列出对应的行业标准与链接（GitHub 官方文档、EditorConfig 规范、Martin Fowler 用户故事等）
- **可裁剪**：配置文件模板为多语言/多技术栈超集，按项目实际技术栈保留对应行，删除无关行（参见各配置文件模板中的裁剪说明）
- **Agent 友好**：所有描述面向 Agent 可理解的规则式语言；红线约束明确可执行；文档生成工作流四阶段清晰可追溯
